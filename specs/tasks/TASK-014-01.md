# TASK-014-01 — Create GitHub Actions Multiplatform Release Workflow

## Status
`DONE`

## Linked Design
DES-014 — Release Management & CI/CD Build System

## Description
Create `.github/workflows/release.yml` that triggers on `v*` tag pushes, builds Cubi Frontend for Windows, macOS (Intel + ARM), and Linux in parallel, and uploads all installers to a GitHub Release.

## Acceptance Criteria
- [ ] Workflow file exists at `.github/workflows/release.yml`
- [ ] Triggers on tags matching `v[0-9]+.[0-9]+.[0-9]+*`
- [ ] Matrix includes: windows-latest, macos-latest (x64), macos-latest (arm64), ubuntu-22.04
- [ ] Ubuntu job installs WebKit2GTK and other Linux dependencies
- [ ] Rust toolchain installed with correct cross-compile targets for macOS
- [ ] Rust cache configured to speed up subsequent builds
- [ ] `npm ci` used (not `npm install`) for reproducible installs
- [ ] `tauri-apps/tauri-action@v0` runs with correct `args` per platform
- [ ] GitHub Release created with tag name and version in title
- [ ] `GITHUB_TOKEN` permissions set to `contents: write`

## Implementation Notes
- macOS arm64 and x64 share the same `macos-latest` runner but pass different `--target` args
- Linux runner must install: `libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf`
- `swatinem/rust-cache@v2` must use `workspaces: './src-tauri -> target'`
