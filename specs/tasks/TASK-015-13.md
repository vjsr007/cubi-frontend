## Description
Implement the PC Scraper Settings React component and integrate it into ScraperPage as a "PC Games" tab. Shows API key fields, source toggles, tool status, and a job runner with progress.

## Acceptance Criteria

### Component: `PcScraperSettings.tsx`
- [ ] **API Keys section**: fields for IGDB Client ID, IGDB Client Secret, SteamGridDB API Key (shared with existing scraper config), MobyGames API Key, YouTube API Key
- [ ] **Tools Status section**: reads from `checkPcScraperTools()` on mount
  - Shows green check / red X for yt-dlp and Chrome
  - Shows detected path if found
  - Link to yt-dlp install instructions if not found
- [ ] **Source Priority section**: checkboxes for each `PcMetadataSource`
  - Ordered: Steam Store (always on, no key needed), IGDB, SteamGridDB, MobyGames, PCGamingWiki (always on), YouTube, Web Scraper
  - Lock icon on sources that require missing credentials
- [ ] **Media Options section**:
  - Max screenshots slider (1-10, default 5)
  - "Download trailers locally" toggle (disabled + tooltip if yt-dlp not found)
  - "Use headless browser" toggle (disabled + tooltip if Chrome not found)
- [ ] **Run Job section**:
  - Dropdown: "All PC games" / "Missing metadata only" / "Missing media only"
  - "Overwrite existing" checkbox
  - "Enrich PC Games" button → calls `runPcMetadataJob`
  - Real-time progress bar + current game name (listens to `scrape-progress` Tauri event)
  - Cancel button (calls `cancelScrapeJob`)
- [ ] Config auto-saved on blur (no Save button needed)
- [ ] Error display for failed job

### Integration
- [ ] `ScraperPage.tsx` gains a tab bar: "Scrapers" (existing) | "PC Games" (new)
- [ ] Switching tabs shows the correct panel
- [ ] Tab state persisted in component (not URL)
