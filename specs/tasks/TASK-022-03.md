# TASK-022-03: Catalog service (sync, import, ownership matching)

## Context
Implement the core catalog business logic: local DAT file import, remote DAT download (with ZIP support), ownership matching (exact filename + normalized stem fallback), and config management.

## Parent Design
- **DES**: [DES-022 — Game Catalog Database](../design/DES-022-game-catalog-database.md)

## Acceptance Criteria
- [x] `import_dat_file()` auto-detects format (XML vs text), parses, bulk inserts, runs ownership matching
- [x] `match_ownership()` uses exact filename match (primary) + normalized stem match (fallback)
- [x] `refresh_ownership()` re-runs matching for one or all systems
- [x] `download_and_import_dat()` downloads DAT from URL, handles ZIP extraction, imports via temp file
- [x] `extract_dat_from_zip()` finds first .dat/.xml in ZIP archive
- [x] `get_default_dat_urls()` returns hardcoded mapping of system_id → libretro-database DAT URLs for 27 systems
- [x] `get_catalog_config()` and `set_download_url()` for TOML config management
- [x] Catalog DB CRUD: `bulk_insert_catalog_games`, `upsert_catalog_sync`, `get_catalog_stats`, `get_catalog_games` (paginated), `update_catalog_ownership`, `get_catalog_system_ids`, `get_catalog_syncs`

## Implementation Notes
- Ownership matching: Since `games` table has no SHA-1 hashes, matching uses filename stems
- Download: reqwest client with 120s timeout, 50 MB limit, ZIP magic byte detection
- Bulk insert: Single SQLite transaction with prepared statements for performance
