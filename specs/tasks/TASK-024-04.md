# TASK-024-04: GOG cloud service — owned games via Galaxy local token

## Context
Reads GOG Galaxy's authentication token from its local SQLite database, fetches the user's owned game IDs from `embed.gog.com/user/data/games`, resolves titles from the products API, and merges with locally-installed GOG game paths.

## Parent Design
- **DES**: [DES-024 — Store Cloud Library Design](../design/DES-024-store-cloud-library.md)

## Acceptance Criteria
- [ ] GOG Galaxy SQLite DB is opened read-only from `%LOCALAPPDATA%\GOG.com\Galaxy\storage\galaxy-2.0.db`
- [ ] Auth token extracted: `SELECT token FROM Authentication LIMIT 1` (or appropriate table/column)
- [ ] Game IDs fetched from `https://embed.gog.com/user/data/games` with `Cookie: gog-al=<token>`
- [ ] Titles + box art resolved via `https://api.gog.com/products?ids=<csv>&expand=downloads`
- [ ] Games with matching install path detected as `installed: true`
- [ ] Uninstalled games get `file_path = "goggalaxy://openGameView/<productId>"`
- [ ] Results cached with store = `"gog"`, TTL = 24 h
- [ ] Graceful fallback to local-scan-only if DB file absent or token invalid

### GOG DB path
```
%LOCALAPPDATA%\GOG.com\Galaxy\storage\galaxy-2.0.db
```
Open with `rusqlite::Connection::open_with_flags(path, OpenFlags::SQLITE_OPEN_READ_ONLY)` to avoid locking.

### API endpoints
```
# Owned game IDs (cookie auth)
GET https://embed.gog.com/user/data/games
Cookie: gog-al=<access_token>

# Product details batch
GET https://api.gog.com/products?ids=123456,789012&expand=downloads&locale=en-US

# Box art from response:
product.images.logo2x  or  product.images.background
```
