# TASK-022-09: Post-scan ownership refresh hook

## Context
After the ROM scanner completes, automatically refresh catalog ownership matching so that newly discovered ROMs are correctly marked as "owned" in the catalog without requiring a manual refresh.

## Parent Design
- **DES**: [DES-022 — Game Catalog Database](../design/DES-022-game-catalog-database.md)

## Acceptance Criteria
- [x] After `scanLibrary()` completes successfully in `libraryStore.ts`, call `refreshCatalogOwnership()` automatically
- [x] The refresh runs silently (no toast) — only log to console
- [x] If catalog is empty (no syncs), the refresh call is skipped to avoid unnecessary work
- [x] The refresh does not block the scan completion UI (runs in background after scan toast)

## Implementation Notes
The scan completion flow is in `src/stores/libraryStore.ts` → `scanLibrary()`. After the scan result is processed and `loadSystems()` is called, add a fire-and-forget call to `api.refreshCatalogOwnership()`.

### Files to Create/Modify
| File | Action | Description |
|------|--------|-------------|
| `src/stores/libraryStore.ts` | MODIFY | Add catalog ownership refresh after scan completion |

### Key Code Pattern
```typescript
// In scanLibrary(), after loadSystems():
// Fire-and-forget — don't await, don't block scan UI
api.refreshCatalogOwnership().catch(e => console.warn('Catalog ownership refresh:', e));
```
