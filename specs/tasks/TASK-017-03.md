# TASK-017-03: Backend Tauri Commands for Emulator Settings

## Context
Expose the emulator settings service to the frontend via Tauri IPC commands. These commands allow the React UI to query setting definitions, read/write per-emulator setting values, reset to defaults, and preview generated config snippets.

## Parent Design
- **DES**: [DES-017 — Emulator General Settings](../design/DES-017-emulator-settings.md)

## Acceptance Criteria
- [x] `get_setting_definitions` command — returns all `SettingDefinition` records from DB
- [x] `get_config_writers_info` command — returns `ConfigWriterInfo` for all 8 emulators (no DB dependency)
- [x] `get_emulator_settings` command — returns stored `EmulatorSettingValue` list for a given emulator
- [x] `get_all_emulator_settings` command — returns all stored setting values across all emulators
- [x] `set_emulator_setting` command — saves a single setting value (emulator_name, setting_key, value)
- [x] `reset_emulator_settings` command — deletes all stored settings for a given emulator (reverts to defaults)
- [x] `preview_emulator_config` command — returns the generated config snippet string for an emulator
- [x] All 7 commands registered in `lib.rs` invoke handler

### Key Code Patterns
- All commands follow existing pattern: `State<'_, Database>` for DB access
- `get_config_writers_info` is the only command that does not need DB state (writers are statically defined)
- All commands return `Result<T, String>` for IPC serialization
- Service-layer delegation: commands are thin wrappers that call `emulator_settings_service` functions
