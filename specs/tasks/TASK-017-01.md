# TASK-017-01: Backend Setting Definitions Model, DB Schema, and Service

## Context
Create the foundational data model for canonical emulator settings: typed setting definitions with categories, a DB schema to persist them, and the core service that seeds definitions, merges stored values with defaults, and enforces locked settings (e.g. V-Sync always OFF).

## Parent Design
- **DES**: [DES-017 — Emulator General Settings](../design/DES-017-emulator-settings.md)

## Acceptance Criteria
- [x] `SettingType` enum with `Bool`, `Select`, `Range` variants (serde-serializable)
- [x] `SettingCategory` enum with `Video`, `Audio`, `System`, `Performance` variants
- [x] `SettingDefinition` struct with key, display_name, description, setting_type, options, range_min/max, default_value, category, sort_order, locked flag
- [x] `EmulatorSettingValue` struct with emulator_name, setting_key, value
- [x] `ConfigWriterInfo` struct with emulator_name, config_format, supported_settings, default_config_path
- [x] 10 canonical setting constants (`SETTING_INTERNAL_RESOLUTION`, `SETTING_FULLSCREEN`, `SETTING_VSYNC`, etc.)
- [x] `all_setting_definitions()` function returning all 10 canonical definitions

- [x] DB table `emulator_settings` (emulator_name + setting_key composite PK, value)
- [x] DB methods: `upsert_setting_definition`, `get_setting_definitions`, `get_emulator_settings`, `get_all_emulator_settings`, `set_emulator_setting`, `delete_emulator_settings`

- [x] V-Sync setting is locked — `set_setting()` rejects changes to locked keys

### Key Design Decisions
- **Canonical settings pattern**: 10 abstract setting keys that get mapped to each emulator's native config format by config writers (TASK-017-02)
- **Locked settings**: V-Sync is always OFF to prevent input lag; the `locked` flag on `SettingDefinition` makes this enforced at the service layer
- **Merge strategy**: `get_merged_settings()` combines DB-stored overrides with canonical defaults, respecting locked values and filtering to only settings the emulator's config writer supports
