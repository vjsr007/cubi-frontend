## Description
Implement a search service that uses DuckDuckGo Instant Answer API (free, no auth) to find a game's official website and other relevant pages (store pages, review pages). Includes heuristics to identify the best URL from search results.

## Acceptance Criteria
- [ ] `find_official_site(title: &str) -> Result<Option<String>, String>`
  - Query: `https://api.duckduckgo.com/?q={encoded_title}+PC+game+official+site&format=json&no_html=1&skip_disambig=1`
  - Extracts `AbstractURL` and `Results[].FirstURL`
  - Applies domain scoring heuristics (see below)
  - Returns highest-scored URL or `None`
- [ ] `search_for_page(query: &str) -> Result<Vec<SearchResult>, String>`
  - Generic search returning up to 5 `SearchResult { title, url, snippet }`
  - Used by orchestrator to find store pages for unknown games
- [ ] **Domain scoring heuristics** for `find_official_site`:
  - Score +10: URL contains normalized game title
  - Score +5: domain is `.com`, `.gg`, `.io`, `.net`, `.org`
  - Score +3: domain is `store.steampowered.com`, `gog.com`, `epicgames.com`
  - Score -5: domain is `gamespot.com`, `ign.com`, `metacritic.com`, `imdb.com`
  - Score -10: domain is `wikipedia.org`, `fandom.com`, `wikia.com`
- [ ] `normalize_title(title: &str) -> String` — shared utility (lowercase, strip articles, punct)
  - This must be the same function used by IGDB and MobyGames services (move to `services/mod.rs`)
- [ ] Request timeout: 8s
- [ ] `SearchResult` struct: title, url, snippet
