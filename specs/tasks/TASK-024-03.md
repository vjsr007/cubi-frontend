# TASK-024-03: Epic cloud service — owned games via local OAuth token

## Context
Reads the Epic Games Launcher's locally-stored OAuth access token (no in-app login required) to call the entitlements API and return the full owned library, merged with locally-installed manifest data.

## Parent Design
- **DES**: [DES-024 — Store Cloud Library Design](../design/DES-024-store-cloud-library.md)

## Acceptance Criteria
- [ ] Epic OAuth token is read from `%LOCALAPPDATA%\EpicGamesLauncher\Saved\Config\Windows\GameUserSettings.ini` (section `[RememberMe]`)
- [ ] If token file is absent or cannot be parsed → return local-scan-only, no error/panic
- [ ] Entitlements API is called with `Authorization: Bearer <token>`
- [ ] Each game is marked `installed: true` if a matching `.item` manifest exists in Epic's Manifests folder
- [ ] Uninstalled games have `file_path = "com.epicgames.launcher://apps/<namespaceId>?action=install"`
- [ ] Results written to `pc_cloud_cache` with store = `"epic"`, TTL = 24 h
- [ ] `force_refresh: false` + fresh cache → no API call

### Token file path
```
%LOCALAPPDATA%\EpicGamesLauncher\Saved\Config\Windows\GameUserSettings.ini
```
Look for section `[RememberMe]`, key `RememberMeData` (JSON string containing `accessToken`).

Alternative path checked by Heroic Games Launcher:
```
%APPDATA%\EpicGamesLauncher\Data\Saved\Config\Windows\GameUserSettings.ini
```

### Entitlements endpoint (reference: Heroic / Legendary)
```
GET https://launcher-public-service-prod06.ol.epicgames.com/launcher/api/public/assets/Windows?label=Live
Authorization: Bearer <token>
```
