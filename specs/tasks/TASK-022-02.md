# TASK-022-02: DAT file parsers (No-Intro XML + Redump text)

## Context
Implement two parsers for the industry-standard DAT file formats used by No-Intro (XML) and Redump (ClrMamePro text). Both return `Vec<CatalogGame>` plus DAT name and version metadata.

## Parent Design
- **DES**: [DES-022 — Game Catalog Database](../design/DES-022-game-catalog-database.md)

## Acceptance Criteria
- [x] `parse_nointro_dat()` parses No-Intro XML via quick-xml, extracting header (name, version) and game/rom entries
- [x] `parse_redump_dat()` parses ClrMamePro text format line-by-line with state machine
- [x] Both parsers extract: title, rom file_name, size, crc32, md5, sha1
- [x] Region extraction from title parenthesized segments (e.g., "(USA)" → "USA")
- [x] Deterministic ID generation via BLAKE3 hash of `system_id|dat_name|file_name`
- [x] Helper `extract_region()` handles 19 known region names
- [x] Helper `normalize_stem()` strips regions, revisions, brackets, articles for fuzzy matching
- [x] ROM line parser handles quoted and unquoted values in ClrMamePro format

## Implementation Notes
- No-Intro parser follows the quick-xml reader pattern from `gamelist_service.rs`
- Redump parser uses a simple line-based state machine (no regex)
- BLAKE3 used instead of SHA-256 since it's already a project dependency
