# TASK-015-02: Tauri Commands for ROM Path CRUD

## Context
Expose the DB CRUD operations and system registry as Tauri IPC commands so the frontend can manage ROM path overrides.

## Acceptance Criteria
- [x] `get_system_registry_list` command returns all known systems as `Vec<SystemDefInfo>`
- [x] `get_rom_path_overrides` command returns all overrides as `Vec<RomPathOverride>`
- [x] `set_rom_path_override` command validates path (exists + is_dir) before saving
- [x] `delete_rom_path_override` command removes an override
- [x] All 4 commands registered in `lib.rs` `generate_handler![]`
- [x] `SystemDefInfo` and `RomPathOverride` structs are Serialize/Deserialize

## Implementation Notes

### Files Created
- `src-tauri/src/commands/rom_paths.rs` — 4 commands + 2 structs

### Files Modified
- `src-tauri/src/commands/mod.rs` — Added `pub mod rom_paths`
- `src-tauri/src/lib.rs` — Registered commands in handler macro

### Key Design Decisions
- Path validation in `set_rom_path_override`: returns descriptive error strings ("Path does not exist", "Path is not a directory")
- `get_system_registry_list` maps from internal `SystemDef` to `SystemDefInfo` (IPC-safe DTO)
- Commands access DB via `tauri::State<AppState>` pattern

## Test Plan

- Verify error message on non-existent path
- Verify error message on file (not directory) path
