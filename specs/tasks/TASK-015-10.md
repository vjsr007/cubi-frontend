# TASK-015-10 — PC Metadata Orchestrator

## Status
`TODO`

## Description
Implement `pc_metadata_orchestrator.rs` — the central coordinator that runs all PC metadata sources in priority order, filling only missing fields unless `overwrite=true`. Returns a `GameInfoPatch` with only changed fields to minimize DB writes.

## Acceptance Criteria
- [ ] `enrich_pc_game(game, config, dest_dir, overwrite, cancel, emit_progress) -> Result<GameInfoPatch, String>`
- [ ] `GameInfoPatch` struct: all GameInfo fields as `Option<T>` — `None` means "no change"
- [ ] Priority pipeline (see DES-015 § 3.8):
  1. Steam Store API → if `game.source == "steam"` and steam_appid present
  2. IGDB → if configured AND (description missing OR overwrite)
  3. SteamGridDB Extended → if api_key configured
  4. MobyGames → if configured AND description still missing
  5. PCGamingWiki → always (free)
  6. YouTube → if configured OR yt-dlp present
  7. Web Scraper → if `website` field known
- [ ] Each step skips if all its target fields are already filled (respects `overwrite` flag)
- [ ] Each step wrapped in `catch_unwind` equivalent — a single source failure does NOT abort the pipeline
- [ ] Cancellation check (`cancel.load(Relaxed)`) before each source
- [ ] Progress events emitted after each source: `{ current, total, game_title, status: "Fetching Steam Store...", ... }`
- [ ] `run_pc_metadata_job(job: PcScrapeJob, app: AppHandle) -> Result<ScrapeResult, String>`
  - Loads all PC games from DB (or specific IDs from job)
  - Calls `enrich_pc_game` per game
  - Writes patch to DB after each game
  - Emits `scrape-progress` event per game
  - Respects `cancel` flag from `app.state::<Arc<AtomicBool>>()`
