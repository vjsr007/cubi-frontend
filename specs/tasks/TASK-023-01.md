# TASK-023-01: Database Schema + AppConfig Model Expansion

## Context
Extend the SQLite database with `emulator_preferences` table to persist per-system emulator selections, and update the Rust config models to include the preference storage field. This is the foundation for all subsequent tasks.

## Parent Design
- **DES**: [DES-023 — Multi-Emulator Per-System Support Design](../design/DES-023-multi-emulator-system.md)

## Acceptance Criteria
- [ ] `emulator_preferences` SQLite table created with columns: `system_id (PRIMARY KEY)`, `selected_emulator (TEXT NOT NULL)`, `created_at (TEXT DEFAULT now)`, `updated_at (TEXT DEFAULT now)`
- [ ] Table creation script included in `src-tauri/src/db/mod.rs` initialization function
- [ ] `AppConfig` struct in `src-tauri/src/models/config.rs` includes new field: `emulator_preferences: HashMap<String, String>`
- [ ] `AppConfig::default()` initializes `emulator_preferences` as empty HashMap
- [ ] New Rust models file `src-tauri/src/models/emulator.rs` created with `EmulatorChoice` and `SystemEmulatorChoice` structs
- [ ] Both structs derive `Debug`, `Clone`, `Serialize`, `Deserialize`
- [ ] All Rust files compile without warnings
- [ ] Backward compatibility: existing databases without the new table are upgraded gracefully (migration runs on app start)
- [ ] Database queries use `rusqlite` prepared statements (no SQL injection risks)
- [ ] Added test to verify table creation succeeds

## Implementation Notes
**Guidance for the implementor:**

### Database Migration Strategy
- Modify `src-tauri/src/db/mod.rs` where tables are initialized
- Use `CREATE TABLE IF NOT EXISTS` — safe for first run and upgrades
- Structure:
  ```rust
  pub fn init_db(conn: &Connection) -> Result<(), String> {
      // Existing tables...
      
      // New: emulator preferences table
      conn.execute(
          "CREATE TABLE IF NOT EXISTS emulator_preferences (
              system_id TEXT PRIMARY KEY,
              selected_emulator TEXT NOT NULL,
              created_at TEXT DEFAULT (datetime('now')),
              updated_at TEXT DEFAULT (datetime('now'))
          )",
          [],
      ).map_err(|e| format!("Failed to create emulator_preferences table: {}", e))?;
      
      Ok(())
  }
  ```

### Config Model Updates
- Open `src-tauri/src/models/config.rs`
- In the `AppConfig` struct, add:
  ```rust
  /// Per-system emulator preference (system_id → emulator_name).
  #[serde(default)]
  pub emulator_preferences: HashMap<String, String>,
  ```
- Update `AppConfig::default()` to include: `emulator_preferences: HashMap::new()`
- Update `AppConfig` serialization/deserialization tests if they exist

### New Emulator Models File
- Create `src-tauri/src/models/emulator.rs` with:
  ```rust
  use serde::{Serialize, Deserialize};
  
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct EmulatorChoice {
      pub emulator_name: String,
      pub detected_path: Option<String>,
      pub is_installed: bool,
  }
  
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct SystemEmulatorChoice {
      pub system_id: String,
      pub system_name: String,
      pub available_emulators: Vec<EmulatorChoice>,
      pub selected_emulator: Option<String>,
  }
  ```
- Add module export in `src-tauri/src/models/mod.rs`: `pub mod emulator;`
