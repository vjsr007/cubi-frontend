# TASK-017-02: Backend Config Writers for 8 Emulators

## Context
Implement the Strategy pattern for translating canonical setting keys into each emulator's native config format. Each emulator has its own config file format (cfg, ini, yml, json, toml) and its own key names, value representations, and file paths.

## Parent Design
- **DES**: [DES-017 — Emulator General Settings](../design/DES-017-emulator-settings.md)

- [x] `EmulatorConfigWriter` trait defined with methods: `emulator_name()`, `config_format()`, `supported_settings()`, `default_config_path()`, `preview_config()`
- [x] `RetroArchConfigWriter` — `.cfg` format, maps resolution to integer multiplier, language to numeric code
- [x] `DolphinConfigWriter` — `.ini` format, maps settings to `[Core]`, `[Display]`, `[DSP]` sections
- [x] `Pcsx2ConfigWriter` — `.ini` format, maps to PCSX2-specific keys under `[EmuCore/GS]`, `[EmuCore]`, `[SPU2]`

### Key Design Decisions
- **Strategy pattern**: Each writer is a zero-sized struct implementing `EmulatorConfigWriter`, stored as `Box<dyn EmulatorConfigWriter>` in the factory
- **Preview-only**: Config writers generate preview snippets — they do not write directly to emulator config files (that would risk overwriting user customizations)
- **Value translation**: Each writer has private helper methods (e.g. `resolution_value()`, `language_value()`) to map canonical values to emulator-native representations
- **Supported settings filter**: Each writer declares which of the 10 canonical settings it supports; the UI only shows relevant settings per emulator
