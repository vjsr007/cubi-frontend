## Description
Implement YouTube trailer/gameplay video search using the YouTube Data API v3, with a fallback to Invidious (no key). Download videos locally via yt-dlp subprocess if available.

## Acceptance Criteria
- [ ] `search_trailer(title: &str, api_key: Option<&str>) -> Result<Option<VideoResult>, String>`
  - Primary: YouTube Data API v3 `search.list` endpoint
    - Query: `{title} official trailer game`
    - Type: `video`, maxResults: 3
    - Returns best match `VideoResult { video_id, title, url }`
  - Fallback (no key): Invidious public API
    - `https://vid.puffyan.us/api/v1/search?q={query}&type=video&fields=videoId,title`
- [ ] `search_gameplay(title: &str, api_key: Option<&str>) -> Result<Option<VideoResult>, String>`
  - Same but query: `{title} gameplay PC 2024`
- [ ] `check_ytdlp() -> Option<PathBuf>` — uses `which::which("yt-dlp")` to locate binary
- [ ] `download_video(url: &str, dest_path: &Path) -> Result<(), String>`
  - Calls `yt-dlp` via `tokio::process::Command`
  - Args: `-f "bestvideo[ext=mp4][height<=1080]+bestaudio[ext=m4a]/best[ext=mp4]/best"`, `--merge-output-format mp4`, `-o {dest_path}`, `--no-playlist`, `--socket-timeout 30`
  - Returns error if yt-dlp not found (non-fatal, caller stores URL only)
- [ ] `download_video` timeout: 5 minutes max via `tokio::time::timeout`
- [ ] Dest path: `media/pc/videos/{sanitized_game_name}_trailer.mp4`
